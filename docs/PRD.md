## 0. 제품 감각 (북극성)

기능은 최대한 단순하게. 대신 감정은 매일 조금:

- **타이머** — 미응답 48h 만료 (긴장감)
- **랜덤 뽑기** — 누가 받을지 / 누구에게 올지 모름
- **일일 한 장** — 하루 1회 무료 발송 (희소성)
- **양방향 OK** — unlock 전 설렘 유지
- **짧은 대화** — 매칭 후 무제한 메신저가 아님 (조건 설계 중)

북극성 KPI: **주간 양방향 OK(unlock) 수** + D1 재방문.

## 3. 역할 · 성별 규칙 (가설 · 검토 중)

| 역할 | 누가 | 하는 일 |
|------|------|---------|
| **발송** | **여성만** | 미션/쪽지 작성·전송, 답장 평가(OK/패스) |
| **수행** | **남성만** | 미션 수신·수행/답장, OK/패스 |

**의도:** 남성이 이상한·불쾌한 미션을 뿌리는 리스크를 구조적으로 줄인다.  
**상태:** 제품 가설. 스토어·차별 이슈·성비 수급을 보고 **유지 / 완화(양방향 가능) / 역할 스왑 실험** 재검토.

→ [`decisions/0006-gender-roles-women-send.md`](./decisions/0006-gender-roles-women-send.md)

## 4. 핵심 루프

```text
(여) 미션 작성 → 랜덤 풀 배달 → (남) 수행/답장 → 양쪽 OK|패스
 → 양방향 OK 시 프로필 공유 unlock → 조건부 인앱 채팅
```

| 규칙 | 내용 |
|------|------|
| Unlock 전 | 미션·답장만. 신원 비공개 |
| OK | **양방향**일 때만 프로필 공유 |
| 패스 | 페널티 최소. 동일 쌍 쿨다운 14일 |
| 타이머 | 미응답 **48h** 만료 |
| 답장 | 텍스트·칩 (사진 Phase 2) |

## 5. 발송 · 과금

| 항목 | 규칙 |
|------|------|
| 무료 발송 | **하루 1회** |
| 추가 발송 | **과금** (티켓/부스터) |
| 수신·답장·OK | 무료 유지 (핵심 루프 비오염) |

이상형 필터:

| 필터 개수 | 과금 |
|-----------|------|
| **1개** | 무료 (나이 / 지역 / 키 등 대표 조건 중 택1) |
| **2개+** | 티켓 필요 |

대표 이상형 필드(초안): 나이대, 지역, 키. (확장 가능하나 목록은 짧게 유지)

→ [`decisions/0004-monetization-after-retention.md`](./decisions/0004-monetization-after-retention.md) (v2로 갱신)

## 6. 랜덤 풀 · 매칭

**완전 랜덤이 아니라 “활성 우선 랜덤 + 단계적 확대”.**

1. **1순위 풀:** 최근 **48시간** 접속한 이성(수행자=남)  
2. 후보 부족 시 → 최근 **7일** 접속으로 확대  
3. 그래도 부족 → **30일** / 전역 폴백 (운영 파라미터)  
4. 하드 제외: 차단, 동일 쌍 쿨다운, 일일 수신 캡, 미온보딩  
5. 소프트: 선택한 이상형 1개(무료) 또는 티켓으로 연 추가 조건

→ [`decisions/0002-matching-weak-fit.md`](./decisions/0002-matching-weak-fit.md) (갱신)

## 7. Unlock 후 프로필 공유 — **추가 설계 필요**

매칭(양방향 OK) 완료 시 **프로필을 서로 보여 준다.**  
아직 미정:

| 옵션 | 장점 | 단점 |
|------|------|------|
| A. 앱 자체 미니 프로필 | 단순, 의존 없음 | 작성 피로, 신뢰 약함 |
| B. Palette 프로필 연동 | 이미 있는 신뢰 프로필 재사용 | 계정·브랜드·정책 결합, 초기 분리 원칙과 충돌 |
| C. 하이브리드 | 기본은 자체, 선택적 Palette 가져오기 | 구현·카피 복잡도 |

**현 방침:** MVP는 **A 자체 미니 프로필**로 가고, B/C는 unlock 이후 Phase에서 재검토.  
상세 필드·작성 UX는 별도 설계 세션.

→ [`decisions/0007-post-unlock-profile-open.md`](./decisions/0007-post-unlock-profile-open.md)

## 8. 인앱 메신저 — **조건 설계 필요**

필요: **있음** (외부로 바로 던지지 않음).  
금지: 매칭 후 **무제한·영구** 채팅앱화.

설계 후보(미정, 하나 고를 것):

| 후보 | 규칙 스케치 |
|------|-------------|
| 메시지 수 캡 | unlock 후 N통(예: 20)까지 무료, 이후 티켓 또는 soft close |
| 시간 캡 | 7일 / 48h 대화 윈도우 |
| 턴제 | 서로 번갈아 보내야 함 (도배 방지) |
| 연락처 핸드오프 | 양쪽 동의 시 외부로 이탈, 스레드 종료 |

→ [`decisions/0003-unlock-ephemeral-chat.md`](./decisions/0003-unlock-ephemeral-chat.md) (갱신)

## 9. MVP In / Out

**In**

- 여 발송 / 남 수행 (가설)
- 하루 1회 무료 발송, 추가 과금 훅
- 이상형 필터 1개 무료
- 48h 활성 우선 랜덤 + 확대
- 텍스트·칩 미션, 양방향 OK, 타이머
- 자체 미니 프로필 unlock
- 조건부 인앱 채팅 (캡은 구현 시 임시값 가능)
- 차단·신고·안전 센터

**Out (초기)**

- 공개 피드·팔로우·AI 결 (숨결 레거시)
- 장소 집합 미션
- 사진 미션
- Palette 강제 연동
- 무제한 메신저·광고 피드

## 13. 구현 갭 (코드 vs PRD v0.3)

현재 코드는 양성별 송수신·하루 3발송·이상형 필터 없음.  
다음 구현 배치에서 맞출 것:

- [ ] 발송=여 / 수신=남 가드
- [ ] 일일 무료 1회 + 초과 시 페이월 훅
- [ ] 활성 48h→7d→확대 풀
- [ ] 이상형 1개 무료 UI
- [ ] 채팅 캡 임시값 적용
- [ ] 미니 프로필 필드 확정 후 온보딩 정리
